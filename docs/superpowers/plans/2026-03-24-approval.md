# @workkit/approval Implementation Plan — "Approval-as-Infrastructure"

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development (recommended) or superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** Build a declarative approval system for Cloudflare Workers — define policies in code, gate any action, get a full audit trail on D1. Composable with @workkit/mcp as middleware.

**Architecture:** Policy engine evaluates action descriptors against declarative matchers. Approval requests are Durable Objects with event-sourced state machines. Tokens are Ed25519-signed, single-use, time-bound. Notifications go through Queues with channel adapters. D1 stores the queryable audit projection.

**Tech Stack:** TypeScript, @workkit/do (state machine, event store, alarms), @workkit/d1 (audit), @workkit/queue (notifications), @workkit/crypto (token signing), @workkit/errors, Vitest, bunup

**Spec:** `workkit/docs/superpowers/specs/2026-03-24-approval-design.md`

**Existing patterns:** Follow `workkit/packages/mcp/` for package structure (tests/, src/, vitest.config.ts, bunup.config.ts, tsconfig.json).

- `globMatch(pattern, value)` — simple glob matching for NameMatcher (`*` = any chars)


- [ ] **Step 2: Implement**

`createGuard(config)` — core guard logic:
1. Evaluate policies against action descriptor
2. If no match → return `{ status: 'allowed', reason: 'no-policy-matched' }`
3. If match with `requiredApprovals > 0`:
   a. Generate request ID
   b. Create DO instance with request data
   c. Generate tokens for each approver
   d. Enqueue notifications
   e. Record in D1 audit
   f. Return `{ status: 'pending', requestId, approvers, expiresAt }`
