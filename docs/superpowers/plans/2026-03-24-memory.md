# @workkit/memory Implementation Plan тАФ "Edge-Native Agent Memory"

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development (recommended) or superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** Build a memory system for AI agents on Cloudflare Workers тАФ store facts, retrieve with hybrid search, manage conversations, query temporal snapshots. D1-first with optional Vectorize.

**Architecture:** Factory pattern тАФ `createMemory({ db })` returns a Memory object. Facts are atomic assertions stored in D1 with temporal metadata. Recall combines keyword search + optional cosine similarity + recency decay + metadata filtering. Conversations are separate table with sliding-window and token-budget compaction.

**Tech Stack:** TypeScript, D1 (facts, conversations), KV (caching, optional), Workers AI (embeddings, optional), Vectorize (vector search, optional), @workkit/crypto (encryption), @workkit/errors, Vitest, bunup

**Spec:** `workkit/docs/superpowers/specs/2026-03-24-memory-design.md`

### Task 4: D1 Schema

**Files:** Create src/schema.ts, tests/schema.test.ts

- `getSchema()` тАФ returns SQL DDL for 4 tables: `facts`, `fact_embeddings`, `messages`, `summaries`
- `facts` table: id, text, subject, source, tags (JSON), confidence, encrypted, created_at, valid_from, valid_until, superseded_by, forgotten_at, forgotten_reason, embedding_status, ttl, idempotency_key (UNIQUE)
- `fact_embeddings` table: fact_id (FK), vector (TEXT/JSON), dimensions
- `messages` table: id, conversation_id, role, content, name, metadata (JSON), token_count, created_at, compacted_into
- `summaries` table: id, conversation_id, content, message_range_from, message_range_to, message_count, original_tokens, summary_tokens, created_at
- Indexes on subject, tags, valid_from, forgotten_at, conversation_id, embedding_status

### Task 5: Fact Storage (remember, get, forget, supersede, expire)

**Files:** Create src/facts.ts, tests/facts.test.ts

- `createFactStore(db)` тАФ factory returning fact CRUD operations
- `remember(text, metadata?)` тАФ INSERT fact, handle idempotency key, return Fact
- `rememberBatch(facts)` тАФ batch INSERT
- `get(factId)` тАФ SELECT by id, parse JSON fields (tags)
- `forget(factId, reason?)` тАФ UPDATE set forgotten_at, forgotten_reason
- `supersede(oldFactId, newText, metadata?)` тАФ remember new fact + UPDATE old fact (superseded_by, valid_until)

- All return `{ ok, value/error }` result pattern

Tests use D1 mock (same pattern as approval audit).

### Task 6: Search (keyword filtering + pagination)

**Files:** Create src/search.ts, tests/search.test.ts

- `createSearch(db)` тАФ factory
- `search(query, options?)` тАФ SELECT facts WHERE text LIKE terms, apply filters (subject, tags, timeRange, includeSuperseded, includeForgotten), paginate with limit/offset, order by chosen field
- Tag filtering uses `json_each()` for array containment
- Temporal filtering: `valid_from <= now AND (valid_until IS NULL OR valid_until > now) AND forgotten_at IS NULL`

---

### Task 7: Recall (hybrid retrieval + scoring)

**Files:** Create src/recall.ts, tests/recall.test.ts

- `createRecall(db, options?)` тАФ factory, options include embeddings config
- `recall(query, options?)`:
  1. Extract search terms
  2. D1 keyword search for candidates (LIKE queries)
  3. If embeddings available: compute query embedding, load candidate embeddings, cosine similarity
  4. Score each candidate: `0.6*similarity + 0.2*recency + 0.1*confidence + 0.1*metadataBonus`
  5. Deduplicate supersession chains (keep most recent)
  6. Filter below threshold, sort by score, limit
  7. Return RecallResult[]

Tests with mock D1 + mock embedding function.
