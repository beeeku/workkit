What's the primary goal of the workkit website?

This shapes everything — architecture, content priority, and what we build first

A

Developer Adoption

Get developers to npm install @workkit/*. Focus on docs, examples, quick start guides, and "why workkit" messaging. The playground and comparisons serve as proof points.

B

Portfolio Showcase

Establish you as a serious open source developer. Beautiful design, live demos, and technical depth signal credibility. Docs matter but the wow-factor matters more.

TanStack-level quality: excellent docs that developers actually use, PLUS a design that makes people share it. The site itself demonstrates the ecosystem's quality.
