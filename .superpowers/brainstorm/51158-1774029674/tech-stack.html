What framework should power the site?

You want to dogfood workkit — so the site should use @workkit/* packages. The question is what sits on top.

A

Astro + Starlight (docs) + custom pages

Starlight handles docs/guides with search, sidebar, i18n. Custom Astro pages for landing, playground, comparisons. Deploys to Cloudflare Pages. Uses @workkit/astro integration.

Pros

  • Best-in-class docs out of the box (TanStack uses this)
  • Your 16 markdown docs work immediately
  • Dogfoods @workkit/astro
  • Static + SSR hybrid on Cloudflare Pages

Cons

  • Starlight has opinions — customization requires overrides
  • Playground needs a custom island component
B

Hono + custom everything (full dogfood)

Build the entire site as a Cloudflare Worker with Hono. Render docs from markdown, build your own playground, use @workkit/hono, @workkit/kv (caching), @workkit/d1 (analytics), @workkit/r2 (assets).

Pros

  • Maximum dogfooding — uses 8+ workkit packages
  • The site IS the demo ("built with workkit")
  • Full control over everything
  • Shows what's possible with the ecosystem

Cons

  • Build docs infrastructure from scratch
  • No search, sidebar, i18n for free
  • Significantly more work upfront
C

Astro Starlight (docs) + Hono Worker (API/playground backend)

Best of both: Starlight for docs, custom Astro pages for landing/comparisons. A separate Hono Worker powers the playground, live demos, and analytics — dogfooding the full workkit stack.

Pros

  • Docs are excellent AND the backend is a showcase
  • Playground API uses @workkit/hono, kv, d1, r2, etc.
  • "Docs powered by Starlight, playground powered by workkit"
  • Ship docs fast, iterate on playground separately

Cons

  • Two deployments to manage
  • Slightly more infra complexity
