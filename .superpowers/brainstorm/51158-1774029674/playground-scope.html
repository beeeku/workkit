What should the playground do?

The playground is the wow-factor. But scope matters — pick what ships first.

A

Code Editor + Run (like StackBlitz/CodeSandbox)

Monaco editor with workkit packages pre-loaded. Users write Worker code, hit Run, and see the response. Code executes on a real Cloudflare Worker backend with actual bindings (KV, D1, R2).

  • Full editor experience with autocomplete
  • Real bindings (sandboxed per session)
  • Share snippets via URL
B

Guided Examples (like Svelte REPL)

Pre-built examples with editable code. Each example demonstrates one package. Users tweak values and see results. Before/after toggle shows raw Cloudflare API vs workkit.

  • Curated, always-working examples
  • Before/after comparison built in
  • Lower complexity to build
  • Each example links to the relevant docs page
C

Guided First → Full Editor Later

Ship guided examples first (each of your 8 existing examples becomes a playground entry). Phase 2 adds a free-form editor. The guided examples become "starter templates" in the full editor.

  • Ships fast — examples already exist
  • Evolves naturally into full playground
  • Guided examples are better for adoption anyway
